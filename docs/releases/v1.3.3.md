# v1.3.3

## Summary

This release hardens the repository in three places that are easy to leave inconsistent in portfolio projects:

- secondary analytical exports
- semantic warehouse coverage
- governance labels between policy and issue intake

## Why It Matters

The core outputs were already strongly governed. This release extends that rigor to secondary exports such as:

- `cac_by_channel.csv`
- `ltv.csv`
- `rfm_segments.csv`
- `cohort_retention.csv`

It also improves the credibility of repository governance by making issue templates use the same label vocabulary documented in the merge policy.

## Technical Deltas

- expanded `src/artifact_validation.py` for secondary CSV contracts
- expanded `tests/test_artifact_validation.py`
- added warehouse semantic coverage for `cohort_retention`
- aligned issue-template labels with the merge policy
- fixed merge-policy text encoding
- enriched incident playbooks with concrete examples
