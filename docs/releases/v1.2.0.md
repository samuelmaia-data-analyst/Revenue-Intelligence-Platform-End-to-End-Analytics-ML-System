# v1.2.0

Release date: 2026-03-19

## Summary

This release focuses on portfolio hardening through clearer governance, stronger downstream validation and better operational readability.

The repository did not add artificial platform scope. It improved trust in what already exists.

## What Changed

- added runtime-health visibility to the Streamlit dashboard
- added downstream warehouse-consumption tests aligned with analytical queries
- removed recurring test-suite warnings from the model training path
- expanded repository and operational documentation
- strengthened repository governance checks around docs and CI expectations

## Why It Matters

This release improves three hiring signals:

1. the repository is easier to audit quickly
2. the warehouse is validated as a consumption layer, not just a persistence target
3. the project looks more maintainable over time, not just well-built once

## Engineering Deltas

- `src/modeling.py` now uses a more stable logistic-regression solver for this context
- `tests/test_warehouse_consumption.py` validates SQL consumption patterns over the warehouse
- `tests/test_repository_governance.py` protects critical docs and CI quality gates from silent regression
- `.gitignore` was tightened for local tooling noise

## Product and UX Deltas

- the Streamlit overview now shows pipeline runtime health, artifact validation and freshness status
- dashboard smoke validation remains part of the CI quality path

## Documentation Deltas

- hiring-review guidance made explicit
- repository structure reflected more accurately in the localized READMEs
- release and troubleshooting documentation expanded
