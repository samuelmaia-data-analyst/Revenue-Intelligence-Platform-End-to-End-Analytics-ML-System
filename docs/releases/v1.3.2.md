# v1.3.2

## Summary

This release improves the weakest remaining portfolio signal after the previous hardening work: evidence of downstream consumption outside the local SQLite query path.

It also makes incident response more concrete and reviewer-facing.

## Why It Matters

The repository already showed:

- an official batch runtime
- warehouse persistence and SQL consumption
- dbt downstream validation
- API and dashboard smoke coverage

What remained less explicit was a consumer path built directly from processed exports as a partner-facing analytical payload. This release adds that surface and documents how to respond when the most likely incident classes occur.

## Technical Deltas

- added `scripts/export_partner_payload.py`
- added `scripts/smoke_partner_payload.py`
- added `docs/incident_playbooks.md`
- added `docs/sql_examples.md`
- expanded CI and local `verify` flow with partner payload validation
- updated README and contribution docs to surface the new downstream and incident material
