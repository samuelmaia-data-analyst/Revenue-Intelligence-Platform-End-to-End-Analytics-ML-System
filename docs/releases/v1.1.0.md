# v1.1.0

Release date: 2026-03-19

## Summary

This release turns the repository from a strong technical portfolio project into a more operationally legible system.

The main changes are:

- modular Streamlit structure with `app/ui` and `app/views`
- processed artifact validation integrated into the pipeline
- dashboard smoke test added to CI
- multilingual README refresh with clearer reviewer reading order
- operational documentation added for runbook, troubleshooting and release discipline

## Why It Matters

This release improves the parts that often make portfolio projects feel intermediate even when the code is solid:

- unclear ownership of the runtime path
- weak operational narrative
- dashboards treated as presentation only, not part of the tested system
- lack of explicit governance for processed outputs

After this release, the repository is easier to inspect, safer to evolve and stronger as a hiring asset.

## Engineering Deltas

- `app/streamlit_app.py` was reduced to a composition shell
- dashboard sections moved into `app/views/dashboard_views.py`
- reusable visual primitives consolidated in `app/ui/primitives.py`
- language dictionaries normalized into `app/dashboard_i18n.py`
- processed artifact contract checks added in `src/artifact_validation.py`
- orchestration now emits `artifact_validation_report.json`
- CI now runs `python scripts/smoke_dashboard.py`

## Operational Deltas

- new runbook added in `docs/runbook.md`
- new troubleshooting matrix added in `docs/troubleshooting_matrix.md`
- lightweight ADR set added under `docs/adr/`
- release process documented in `docs/release_process.md`

## Contract and Behavior Changes

- the pipeline now fails if critical processed CSV or JSON artifacts are missing required schema elements
- the dashboard is now explicitly treated as a smoke-tested consumption layer
- reviewer-facing documentation now points to architecture, runbook, ADRs and troubleshooting in a defined order

## Validation

Validated during this release:

- `python -m pytest -q`
- `python -m ruff check app src tests scripts`
- `python -m black --check app src tests scripts`
- `python -m isort --check-only app src tests scripts`
- `python scripts/smoke_dashboard.py`

## Reviewer Reading Path

If you are opening this repository for the first time, start with:

1. `README.md`
2. `docs/architecture.md`
3. `docs/runbook.md`
4. `docs/troubleshooting_matrix.md`
5. `docs/adr/README.md`

## Portfolio Impact

This release primarily improves:

- perceived maintainability
- operational credibility
- clarity of system ownership
- evidence of architectural decision-making

It does not add artificial platform complexity. It makes the existing system easier to trust.
