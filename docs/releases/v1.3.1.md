# v1.3.1

## Summary

This release focuses on the remaining high-signal gap after the dbt hardening work: proving that curated exports are usable outside the local SQLite warehouse path.

It also tightens reviewer-facing documentation so the repository communicates incident handling and merge discipline more directly.

## Why It Matters

Before this release, the project already validated:

- the batch runtime
- the Streamlit workspace
- the FastAPI surface
- downstream SQL over SQLite
- dbt consumption of the generated warehouse

What was still less explicit was the consumption path for processed CSV/JSON artifacts themselves. This release makes that contract visible and testable.

## Technical Deltas

- added `scripts/smoke_processed_exports.py`
- added CI coverage for processed CSV/JSON export consumption
- expanded the container smoke to assert key processed exports exist
- updated runbook and troubleshooting docs with export, API-container, and dbt incident paths
- updated localized documentation and contribution guidance to include the new smoke surface
