# dbt Layer

This dbt project sits on top of the warehouse generated by the pipeline and is hardened for environment-aware execution.

## Purpose

- formalize analytics-ready models on top of warehouse tables
- expose semantic revenue metrics for finance-grade governance
- add dbt-native tests on the curated layer

## Expected warehouse

Run the pipeline first so the SQLite database exists at:

- `data/warehouse/revenue_intelligence.db`

## Example profile

Copy `dbt/profiles/profiles.yml.example` into your local dbt profiles directory.

The profile is environment-aware through `env_var(...)` and supports:

- `dev`: local SQLite development
- `ci`: deterministic SQLite profile for CI/docs publishing
- `postgres`: Postgres-backed warehouse target for production-style environments

Relevant environment variables:

- `DBT_TARGET`
- `DBT_SQLITE_PATH`
- `DBT_SQLITE_SCHEMA_DIR`
- `DBT_POSTGRES_HOST`
- `DBT_POSTGRES_PORT`
- `DBT_POSTGRES_USER`
- `DBT_POSTGRES_PASSWORD`
- `DBT_POSTGRES_DBNAME`
- `DBT_POSTGRES_SCHEMA`

## Freshness SLAs

Source freshness is explicitly configured in `dbt/models/sources.yml`.

Current SLA for `scored_customers`:

- warning after 12 hours
- error after 24 hours

This is intended to make docs publishing fail fast when warehouse-fed sources become stale.

## Typical commands

```powershell
dbt --project-dir dbt debug
dbt --project-dir dbt source freshness
dbt --project-dir dbt run
dbt --project-dir dbt test
dbt --project-dir dbt docs generate
```
