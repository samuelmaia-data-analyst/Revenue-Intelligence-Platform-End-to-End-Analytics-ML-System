# v1.3.0

## Summary

This release hardens the repository where senior reviewers usually look next:

- downstream dbt validation is now part of the main runtime story
- the API is smoke-tested at the container layer, not only in-process
- localized READMEs document the isolated dbt CLI workflow
- operational docs are more explicit about failure classes and repository breadth

## Why It Matters

Before this release, dbt was already present and technically sound, but its validation path was less visible than the batch core. The API also had smoke coverage, but not yet at the container runtime boundary.

This release closes that gap. The repository now shows that:

- the warehouse produced by the official pipeline supports dbt consumption
- the containerized API exposes a healthy process
- local reproducibility is preserved by isolating the dbt CLI in `.dbt-venv`

## Technical Deltas

- added `scripts/smoke_dbt_sqlite.py`
- added a dedicated `dbt-sqlite` CI job
- aligned dbt docs workflow with `dbt build`
- documented isolated `dbt` CLI setup in localized READMEs
- expanded the runbook with API, dbt, and environment-failure guidance
- made deprecation and top-level breadth policy more explicit
- added a merge-policy document for labels and release gates
